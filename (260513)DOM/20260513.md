# DOM
- 웹 브라우저에서의 javascript : 웹 페이지에서 동적인 기능을 담당.
- 웹 페이지를 구조화된 객체로 제공하여, 프로그램이 언어가 페이지 구조에 접근할 수 있는 방법 제공
- DOM API : 페이지 요소들을 객체 형태로 제공하며 관련된 매서드 제공
- document 객체 : 웹 페이지를 나타내는 DOM 트리의 최상위 객체 / dOM에서 모든 요소는 하나의 객체

# 선택 메서드
- document.querySelector(selector) : 요소 한개 선택, selector와 일치하는 첫 번째 요소를 하나 선택
- document.querySectorAll(selector) : 요소 여러 개 선택
```javascript
const h1Tag = document.querySelector('.heading')
console.log(h1Tag.classList)
```

### 속성 조작
- element.classList.add() : 지정된 클래스 값을 추가
- element.classList.remove() : 지정된 클래스 값을 제거
- element.classList.toggle() : 클래스가 존재한다면 제거하고 false를 반환, 존재하지 않으면 추가하고 true 반환
- 클래스 속성 조작
```javascript
// 1. HTML 문서에서 클래스 이름이 'heading'인 첫 번째 요소를 찾아서 h1Tag 변수에 할당합니다.
const h1Tag = document.querySelector('.heading');

// 현재 h1Tag 요소가 가지고 있는 클래스 목록(DOMTokenList)을 콘솔에 출력합니다.
// 예: DOMTokenList ["heading"]
console.log(h1Tag.classList);

// 2. h1Tag 요소의 클래스 목록에 'red'라는 새로운 클래스를 추가합니다.
// (보통 CSS에 미리 작성된 .red 스타일을 적용하기 위해 사용합니다.)
h1Tag.classList.add('red');
// 'red'가 성공적으로 추가되었는지 콘솔에서 확인합니다.
// 예: DOMTokenList ["heading", "red"]
console.log(h1Tag.classList);

// 3. h1Tag 요소의 클래스 목록에서 'red'라는 클래스를 삭제(제거)합니다.
h1Tag.classList.remove('red');
// 'red'가 삭제되어 원래 상태로 돌아왔는지 콘솔에서 확인합니다.
// 예: DOMTokenList ["heading"]
console.log(h1Tag.classList);

// 4. 'red' 클래스가 요소에 없으면 추가하고, 있으면 제거하는 '토글(Toggle)' 기능을 수행합니다.
// 바로 윗줄에서 'red'를 제거했기 때문에, 현재는 클래스가 없는 상태입니다. 따라서 'red'가 다시 추가됩니다.
h1Tag.classList.toggle('red');
// 'red'가 다시 추가된 최종 상태를 콘솔에서 확인합니다.
// 예: DOMTokenList ["heading", "red"]
console.log(h1Tag.classList);
```
#### 일반 속성 조작 메서드
- Element.getAttribute() : 해당 요소에 지정된 값을 반환(조회)
- Element.setAttribute(name, value) : 지정된 요소의 속성 값을 설정
- Element.removeAttribute() : 요소에서 지정된 이름을 가진 속성 제거

#### DOM 요소 조작 메서드
- document.createElement(tagName) : 작성한 tagName의 HTML 요소를 생성하여 반환
- Node.appendChild() : 한 Node를 특정 부모 Node의 자식 NodeList 중 마지막 자식으로 삽입
- Node.removeChild() : DOM에서 자식 Node를 제거, 제거된 Node를 반환


### 용어 정리 
- Node : DOM의 기본 구성 단위
- NodeList : DOM 메서드를 사용해 선택한 Node의 목록
- Element : DOM 트리에서 HTML 요소를 나타내는 특별한 유형의 Node
- Parsing : 브라우저가 문자열을 해석하여 DOM Tree로 만드는 과정
- 세미콜론 : JS는 문장의 마지막 세미콜론을 선택적으로 사용가능하지만, ASI에 의해 자동으로 세미콜론이 삽입됨.
- 함수 스코프 : 함수의 중괄호 내부를 가리킴.
- 호이스팅 : 코드가 실행 되기 전에, 선언문이 스코프의 최상단으로 끌어올려지는 듯한 현상
- 일시적 사각지대 : 스코프 싲가부터 변수의 초기화가 이루어지는 지점까지의 구간(let, const등은 호이스팅이 되지만, 이 TDZ에 갇혀 있기 때문에 선언 전에 접근할 수는 없음.)
